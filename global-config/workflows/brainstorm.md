description: Brainstorm and design before implementing. Use when starting a new feature or solving a complex problem.
---

## Steps

1. **Understand the problem** — Read the user's request carefully. Ask clarifying questions if needed.
2. **Research existing code** — Search for related files, patterns, and dependencies.
3. **Design the solution** — Propose 2-3 approaches with trade-offs.
4. **Get approval** — Present the design to the user and wait for feedback.
5. **Create implementation plan** — Write a detailed plan after approval.

> **Note:** Do NOT write any production code during brainstorming. Design first, code later.
