description: Review completed code against the original plan and coding standards.
---

## Steps

1. **Load context** — Read the original plan and the completed implementation.
2. **Plan alignment** — Compare implementation against planned approach:
   - All planned functionality implemented?
   - Any unjustified deviations?
3. **Code quality** — Check:
   - Error handling and type safety
   - Naming conventions and organization
   - Test coverage
   - Security and    performance
4. **Architecture** — Verify:
   - SOLID principles
   - Separation of concerns
   - Integration with existing systems
5. **Report issues** — Categorize as:
   - 🔴 **Critical** — Must fix before merge
   - 🟡 **Important** — Should fix
   - 🟢 **Suggestion** — Nice to have
6. **Acknowledge good work** — Always mention what was done well before highlighting issues.
