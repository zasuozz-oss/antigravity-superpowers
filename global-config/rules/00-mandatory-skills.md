# Mandatory Skills Usage

## Critical Rule

**BEFORE any implementation, design, or problem-solving work:**

You MUST invoke the appropriate skill. This is not optional.

## Skill Invocation Requirements

### 1. Brainstorming (MANDATORY)
**Invoke BEFORE:**
- Any new feature implementation
- Any architectural decision
- Any design work
- Any creative problem-solving

**Never skip this.** Even "simple" tasks benefit from structured thinking.

### 2. Test-Driven Development (MANDATORY)
**Invoke BEFORE:**
- Writing any production code
- Implementing any feature
- Adding any functionality

**Write tests first.** No exceptions.

### 3. Systematic Debugging (MANDATORY)
**Invoke WHEN:**
- Any test fails
- Any bug is discovered
- Any unexpected behavior occurs
- Any error needs investigation

**Find root cause first.** No quick fixes.

## Enforcement

If you think "this is too simple for a skill," you are wrong. Invoke the skill.

If you think "I can do this faster without the skill," you are wrong. Invoke the skill.

If you think "I already know what to do," you are wrong. Invoke the skill.

## The Rule

**Even a 1% chance a skill applies = MUST invoke the skill.**

No rationalization. No shortcuts. No exceptions.
